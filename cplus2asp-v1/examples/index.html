Examples

We used several examples as benchmarks of performance and analysis of the C+ to ASP translation framework. Listed below are all of the sample domains, along with downloadable files for Cplus2ASP-compatible versions of each.

To run the examples using Cplus2ASP, install all the tool chain software mentioned on the main page, then simply call Cplus2ASP and pass it the file name(s) of the file(s) associated with the example, optionally specifying which query to run. For example, for Monkey & Bananas, the command line to run the formalization with the classic "monkey gets the bananas" query is as follows:
cplus2asp monkey.cp query=2
The results should look like this:
Solution 1:
0: loc(bananas)=l2 loc(box)=l3 loc(monkey)=l1
ACTIONS: walk(l3)
1: loc(bananas)=l2 loc(box)=l3 loc(monkey)=l3
ACTIONS: pushBox(l2)
2: loc(bananas)=l2 loc(box)=l2 loc(monkey)=l2
ACTIONS: climbOn
3: loc(bananas)=l2 loc(box)=l2 loc(monkey)=l2 onBox
ACTIONS: graspBananas
4: hasBananas loc(bananas)=l2 loc(box)=l2 loc(monkey)=l2 onBox
Note that due to the translator module (Cplus2ASP.bin) being in the prototype phase of development, it is not yet able to translate the non-NMCT examples directly without some minor edits to the original domain descriptions.

NMCT Examples

The simple examples from the paper "Nonmonotonic Causal Theories" serve as good baseline domains to verify the functionality of various aspects of the CCalc language. Each covers different properties of causal logic, including associated syntax and constructs.

  • Going to Work:
    going.cp
  • Lifting the Table:
    lifting.cp
  • Monkey & Bananas:
    monkey.cp
  • Pendulum:
    pendulum.cp
  • Publishing Papers:
    publishing.cp
  • Shooting Turkeys:
    shooting.cp

Zoo World and Traffic World

  • Zoo World: The Zoo World is a larger domain referenced in "Representing the Zoo World and the Traffic World in the Language of the Causal Calculator" that is a framework for describing a functioning zoo with people, animals, areas, and cages, including rules for moving animals around and performing basic tricks.
    zoo-actions.cp zoo-animals.cp zoo-landscape.cp zoo-movement.cp zoo-example.cp zoo-test1.cp zoo-test2.cp zoo-test3.cp zoo-test4.cp zoo-test5.cp zoo-test6.cp zoo-example-bigger.cp zoo-bigger-test1.cp
    Running the Zoo World is a bit more complex than the other examples due to its size. Use this command to run the Zoo World:
    cplus2asp zoo-landscape.cp zoo-animals.cp zoo-movement.cp zoo-actions.cp zoo-example.cp zoo-test1.cp
    "zoo-test1.cp" can be replaced with any of the normal test files, and the "Big Cage Shuffle" can be run by using "zoo-example-bigger.cp" and "zoo-bigger-test1.cp" in place of their respective normal files. Cplus2ASP will automatically find the queries for the given test, and will ask which one to run.
  • Traffic World: The Traffic World is similar to the Zoo World size-wise, and acts as a framework for describing vehicles driving along roads and highways.
    traffic.cp scenario1.cp scenario2.cp scenario3.cp scenario_altmerge.cp
    Running the Traffic World is less complex than the Zoo World, only requiring the inclusion of "traffic.cp" and one of the scenario files, like so:
    cplus2asp traffic.cp scenario1.cp
    "scenario1.cp" can be replaced with any of the scenario files. Cplus2ASP will automatically find the queries for the given scenario, and will ask which one to run.
  • Knapsack of Marbles: A version of the knapsack problem, this is a puzzle version of the original problem wherein quantities of differently-sized marbles have to be placed into bags of certain sizes such that everything perfectly fits and no marbles are left out.
    container.cp marblebags.cp
  • Tower of Hanoi: This is a classic puzzle involving moving discs across a set of pegs while following a few simple rules regarding when and where discs can move.
    hanoi.cp

Lua Examples

  • With the addition of Lua support to ASP, we have extended cplus2asp to be able to incorporate this addition. To demonstrate the syntax of using Lua, we have two simple examples. One is the standard Towers of Hanoi except that the move action does not always work (note this is decided at grounding time so this is not truly non-determinism). The other is a simple domain that reads the value of a hypothetical sensor whose value is stored in a file called "sensor".
    cplus2asp_lua_examples.zip
