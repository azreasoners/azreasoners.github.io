The Cplus2ASP system utilizes a tool chain of programs to translate CCalc-style action descriptions into ASP, solve the resulting ASP program, and transform the output from the ASP solver back into CCalc-style solutions. The archive in the Downloads section of the sidebar contains statically-compiled Linux executables for cplus2asp, cplus2asp.bin, f2lp, and as2transition, along with source code and build instructions for each should you wish to compile the programs yourself.

Prerequisites

In addition to the programs contained in the downloadable Cplus2ASP archive, you will also need to install the programs gringo and clasp, available via the link in the sidebar. If you wish to compile cplus2asp.bin from source, you will also need CMake. cplus2asp.bin comes with pre-built lexer and parser sources, but if you want to rebuild them (or make modifications to either), you will need Flex and Bison.

Installing the Cplus2ASP System Using Pre-Compiled Binaries

The Cplus2ASP archive contains pre-compiled Linux binaries for cplus2asp, cplus2asp.bin, f2lp, and as2transition. To install the system using these binaries, run install.sh script as follows:

sudo ./install.sh

This will install all included programs into "/usr/local/bin", and any relevant support files to "/usr/local/lib". Alternatively, to install to a different base directory, pass the desired base directory to the script as an argument. For example to install to your local user directories "/home/me/bin" and "/home/me/lib", call the install script like so:

./install.sh "/home/me"

This will probably eliminate the need for you to have permissions to use the sudo program. Note that you must use an absolute path if you plan to override the default install location.

Compiling the Cplus2ASP System

If the included binaries are not compatible with your system (or you just want to compile Cplus2ASP from source anyway), each of the included programs has a "src" subdirectory that contains its source code along with a README file explaining how to compile and use the software. Note that the Cplus2ASP system has only been tested on Linux and Mac OSX operating systems, and at this time has not been officially ported to Windows.

To compile and install cplus2asp, run these commands from the "cplus2asp/src" directory in the unzipped archive:

make
sudo make install

This will compile cplus2asp and install it into "/usr/local/bin", with helper files installed to "/usr/local/lib/cplus2asp".

To compile and install cplus2asp.bin, you will need to have CMake installed on your system. Assuming it is present, run these commands from the "cplus2asp.bin/src/build" directory in the unzipped archive:

./configure.sh
make
sudo make install

This will compile cplus2asp.bin and install it into "/usr/local/bin".

To compile and install as2transition, run these commands from the "as2transition/src" directory in the unzipped archive:

make
sudo make install

This will compile as2transition and install it into "/usr/local/bin".
