Using Config in Cplus2ASP

Cplus2ASP v3.1 introduces support for adding a configuration file in the root of your $HOME directory. The file has to be explicitly called '.cplus2asp.config' without the quotes. Do note that giving any argument in commandline would override the argument that was given in config file.

The configuration file is modelled after the ini files that are usually used as configuration files in windows programs. The configuration file is not mandatory and the program does not depend on its availability. In am multiuser system where every user has his/her own directory, to use this config file the user needs to have this file in his/her own $HOME directory.

Example: 
	# This is a comment
	# Do not write comment after a key-value pair. It won't be read
	# [cplus2asp] denotes that the next section has cplus2asp program's configuration pairs. 
	# This is the only section that you have to define.
	[cplus2asp]
	language=c+
	mode=static-auto

The preceding configuration file tells the program to function in the mode static-auto (default is incremental) and use the language c+ (This is already a default). Note that only key value pairs can be written in this file. ANy single argument like number of solutions to the program would result in an error(There is no key that takes number of solutions as its value).
