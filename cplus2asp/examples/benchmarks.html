Cplus2ASP v3 Benchmarking Results

In order to test the performance of the Cplus2ASP v2 system, we compared its performance to its predecessors, Cplus2ASP v1 and CCALC 2, as well as COALA, a similar system which implements a subset of the language C+ (C_taid) as well as the language B.

We used a number of large domain variants, including the Traffic World, Spacecraft Integer, Towers of Hanoi, and Ferryman domains in order to perform the tests. Each test was run on an Intel Core 2 Duo 3.0 GHZ CPU with 4 GB of RAM running Ubuntu 11.10. The CCALC 2 tests used Relsat 2.0 as a SAT solver, while the others used the Clingo 3.0.5 suite as an ASP grounder/solver. All tests were directed to find the minimum the length plan satisfying a set of goals.

The table below details the results of the benchmarks and provides the files used for each benchmark (as well as a trace of the benchmark output) and graph displaying the amount of time taken per step on each system. # indicates the length of the minimum length plan.

Domain Files # CCalc 2 Cplus2ASP v1 COALA Cplus2ASP v2
static incr. static incr.
traffic (altmerge) link 11 889.92s 121.63s n/a n/a 108.73s 16.89
blocks-cost (15) link 8 222.8s 77.19s n/a n/a 20.45s 3.55s

hanoi (8/4) link 33 timeout 133.88s 3321.89s 471.89s 192.17s 15.93s
ferryman (10/4) link 33 39.45s 11.25s 49.56s 2.77s 23.96s 0.92s
ferryman (15/4) link 33 1004.26s 124.75s 962.31s 21.05s 352.8s 7.24s

These results indicate a significant increase in performance between Cplus2ASP v2's incremental running mode and Cplus2ASP v1 as well as 3x increase in performance between Cplus2ASP v2 and COALA. The primary advantages Cplus2ASP has over COALA leading to the performance increase focus around the enhanced expressivity of the input language (resulting in a less verbose representation of domains) as well as a streamlined translation with the use of F2LP and standard translation library files.
