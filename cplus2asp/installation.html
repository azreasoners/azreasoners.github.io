Installation Guide

Precompiled Binary Releases

Beginning at Cplus2ASP v2.2, binary releases will be provided periodically for major revisions in order to facilitate a more user-friendly installation process. The release (shown in the table below) provide all neccessary runtime dependencies and encapsulate installation in a simple installation script that will guide you through the process.

Version Binaries
Linux 32bit Linux 64bit Windows 32bit
3.1 link link N/A
2.3 link link N/A

Linux 32bit and 64bit executables require GLIBC 2.2.5, GLIBCXX 3.4.15 and certain boost libraries in order to execute.

After obtaining the appropriate package listed above, Cplus2ASP can be installed by unpacking the package and running the installation script. Before running the installation scripts make sure you have the lua library and boost filesystem library. Type the following into console. The command is distribution specific and works on debian based systems:

In ubuntu 16.04 or newer to install libboost1.54* add "deb http://mirrors.kernel.org/ubuntu trusty main universe" without the quotes to /etc/apt/sources.list and then run sudo apt-get update from terminal before executing the below command. 

sudo apt-get install liblua5.1 libboost1.54-dev libboost-filesystem1.54-dev libboost-regex1.54-dev libboost-program-options1.54 libtbb2

Then use the installation script as follows: 

	tar xvf package-name.tar.gz
	cd package-name
	./install.sh

Alternatively, for a system-wide install, replace the last command with one of: 

	sudo ./install.sh
	su -c "./install.sh"

depending on your distribution.

The default installation directory is determined by the priviledge level the script is run at. If the command is run on a non-root account, it will default to installing the program in the 'bin' and 'lib' subdirectories of your home directory (creating them if neccessary). Otherwise, it will default to /usr/local/bin and /usr/local/lib, which is appropriate for many distributions.

The script will install the appropriate versions of F2LP and Clingo required by Cplus2ASP.

Compiling from Git

If you wish to install Cplus2ASP on your system, you will first need to acquire the distribution package (see above) and then compile and install the package. The following assumes that you've already downloaded the source code and navigated to the subdirectory created (Cplus2ASP-master if you've downloaded and extracted the .zip file, Cplus2ASP otherwise).

Before continuing, verify that your system has the Autotools and Make suites installed (autoconfig, automake, and autoheader), if your system is a Debian based distribution (such as Debian, Ubuntu, or Mint) this can be done by running the command: 

	sudo apt-get install autoconf automake libtool make

Additionally, new versions of Cplus2ASP now depend on the Boost libraries version 1.44+ as well as the Flex, Bison, and RE2C build tools. If you're running Ubuntu 11.10 (Oneiric) or later this can be done by using the command 

	sudo apt-get install libboost-all-dev flex bison re2c

Otherwise, you should see if Boost 1.44+ and the fore-mentioned build tools have packages for your distribution. Alternatively, Boost can be compiled from the source distribution available here.

After verifying that the appropriate dependencies have been installed, you may configure, compile, and install Cplus2ASP by running the following commands, assuming you're running on a Debian based distribution. 

	./bootstrap && make && sudo make install

On other systems, the following may also work (assuming you have access to the Super User account). 

	./bootstrap && make && su -c "make install"

Finally, Cplus2ASP requires, at a minimum, systems F2LP v1.4+ and clingo 4+ to run effectively. For using python and lua code in cplus2asp you need to use clingo compiled with python and lua support. Please refer to clingo documentation for instructions on how to compile this program. These systems can be found at the following locations:

  • F2LP: http://reasoning.eas.asu.edu/f2lp
  • Clingo: http://potassco.sourceforge.net
