F2LP: Computing Answer Sets of First Order Formulas

This page lists a few example descriptions in the language of F2LP. A detailed manual is available here.

 

Example 1:

 

/* Begin F2LP input */

 

% file example1.fof

% domain from 0 to 9

number(0..9).

 

% domain variables X and Y

#domain number(X;Y).

 

r(1).

s(X) <- r(X) & not ?[X]:(p(X,Y) & q(X,Y)) .

 

/* End F2LP input */

 

Run F2LP:

f2lp example1.fof

 

/* Begin F2LP output */

 

number(0..9).

#domain number(X;Y).

r(1) :- true.

s(X) :- r(X), not _new_pred_1(Y).

_new_pred_1(Y) :- p(X,Y), q(X,Y).

true.

:- false.

#hide true.

 

/* End F2LP output */

 

Here "_new_pred_1" is the new predicate introduced by F2LP.

 

Example 2:

 

/* Begin F2LP input */

 

% file example2.fof

% domain {a,b,c}

object(a;b;c).

 

% domain variable X

#domain object(X).

 

p(a).

r(c).

q(b) <- ?[X]:(p(X) & r(X)).

 

% rule in lparse/gringo syntax

r(X) :- p(X).

 

/* End F2LP input */

 

/* Begin F2LP output */

 

object(a;b;c).

#domain object(X).

p(a) :- true.

r(c) :- true.

#domain object(_NV_1).

q(b) :- p(_NV_1), r(_NV_1).

r(X) :- p(X).

true.

:- false.

#hide true.

 

/* End F2LP output */

 

Here "_NV_1" is the new variable introduced which has the same domain as the variable it replaced (X). Also, the rule in lparse/gringo syntax is copied directly to the output. F2LP does not check for parse errors for such rules.

 

 

Example 3:

 

/* Begin F2LP input */

 

% file example3.fof

% domain 0 - 9

number(0..9).

 

% domain variables X and Y

#domain number(X;Y).

 

% rule with strong negation

-p(1).

 

q(2) <- (Y < 2 & r(Y)) | ?[X]:(p(X) & r(X)).

 

 

/* End F2LP input */

 

/* Begin F2LP output */

 

number(0..9).

#domain number(X;Y).

-p(1) :- true.

#domain number(_NV_1).

q(2) :- Y<2, r(Y).

q(2) :- p(_NV_1), r(_NV_1).

true.

:- false.

#hide true.

 

/* End F2LP output */
