Description: Beyond the study of proofs in mathematics, logic has been applied to designing and reasoning about computer hardware and software. One of the prominent applications of logic in computer science is the use of logic for declarative programming.

Answer set programming (ASP) is a recent form of declarative programming oriented towards difficult combinatorial search problems, and has found to be useful in many knowledge-intensive applications. It has emerged from interaction between two lines of research---on the semantics of negation in logic programming and on applications of satisfiability solvers to search problems. The idea of ASP is to represent the search problem we are interested in as a logic program whose intended models, known as ``answer sets'', correspond to the solutions of the problem, and then find them using an answer set solver -- a system for computing answer sets. Applications of ASP include knowledge representation, plan generation, and product configuration problems in artificial intelligence and to graph-theoretic problems arising in VLSI design, in historical linguistics, and in bioinformatics. Fully declarative semantics of ASP makes it easy to write provably correct programs, and allows easy communication between other users of computational methods besides computer scientists. The significance of this new approach is heightened by the multidisciplinary nature of collaborations and new computing areas that are emerging. For example,

  • Automated Product Configuration: An ASP-based technology that has been commercialized ({\tt www.variantum.com/en}).
  • Decision Support for the Space Shuttle: An ASP system has been a part of autonomous control of the Space Shuttle.
  • Inferring Phylogenetic Trees: An ASP-based method for reconstructing a phylogeny for a set of taxa has been applied to historical analysis of languages and of parasite-host systems.
Other applications (showcases) are listed at http://www.kr.tuwien.ac.at/research/projects/WASP/showcase.html

Projects

Students completed non-trivial ASP programming tasks for their term projects.

    Undergraduate Student Projects

  • Jacob Griffin, Paint by Numbers
    Jake formalized in ASP a type of grid puzzle called "Paint by Number", which is to complete a picture directed by constraints given by numbers. He also implemented GUI in C#.net to make input of puzzles easier to test and show the pretty pictures.

  • Tylar Hoag, Wumpus World
    Tylar formalized in ASP a reasonably restricted version of the Wumpus World, in which reasoning has to involve unknown environment.

  • Thomas Lang, Some Variations of Sudoku
    Tom formalized in ASP three variations of Sudoku puzzles, for instance one involving 5 different 5x5 boards with one board overlapping with the other four board.

  • Houston Murphy, Jealous Husbands Puzzle
    Houston's project is to test elaboration tolerance of ASP language by formalizing some elaborations of Jealous Husbands Puzzle, which is similar to Missionaries and Cannibals puzzle. He investigated how much of the code can be reused, each time some elaborations have to taken into account.

  • Matthew Roberson, Using ASP to Solve Binary-Determination Logic Puzzles
    Matt formalized two grid puzzles "Kuromasu" and "Light up" in ASP.

  • Richard Seale, Rubik's Cube
    Richard's project is to formalize the movement of Rubik's cube and analyzed possible configurations of the puzzle using ASP.

    Graduate Student Projects

  • Michael Casolary, Comparing Solver Performance Using Peg Solitaire
    Michael formalized a few Peg Solitaire games and analyzed the performance ASP solvers on this domain. He also formalized the games in C+ and compared the performance with CCalc.

  • Benoit Catherinet, Endgame Chess
    Benoit investigated how to use ASP to prove a small theorems about Chess game, such as detecting checkmate.

  • Juraj Dzifcak, Using CCG and Lambda Calculus together with the New ASP Semantics to Characterize NL sentences
    Juraj investigated how natural language sentences involving existential quantification can be represented in the new language of stable models proposed in [Ferraris, Lee and Lifschitz, 2007].

  • Tae-Won Kim, Formalizing Korean Honorification in ASP
    Honorifics in Korean language has many sophisticated rules. Tae-won formalized these rules as ASP knowledge base, which can be used to infer speaker's relation from the use of honorific markers.

  • Tuan Nguyen, Logic Programming Approach to Partial Satisfaction Planning
    Tuan formalized partial satisfaction planning problem in ASP, in which actions have cost and goals have reward. He applied the splitting theorem to prove if his encoding is correct.
